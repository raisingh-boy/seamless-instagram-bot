
## 2026-05-21T04:50 — Instagram Reel Download Method

**Проблема:** yt-dlp, instaloader, прямые CDN запросы — всё требует Instagram cookies для HD
**Решение (работающее):**
1. Playwright открывает страницу рилса (не важно что там signup overlay)
2. Перехватываем response'ы, ищем scontent-*.cdninstagram.com с .mp4
3. Через page.evaluate() делаем fetch() CDN URL внутри браузера (там есть сессионные куки)
4. arrayBuffer → Uint8Array → Python bytes → save to file

**Схема:**
```python
result = await page.evaluate('''
    async () => {
        const resp = await fetch('https://scontent-...mp4?_nc_cat=...', {
            headers: { 'Range': 'bytes=0-', 'Referer': 'https://www.instagram.com/' }
        });
        const buffer = await resp.arrayBuffer();
        return Array.from(new Uint8Array(buffer));
    }
''')
```

**Результат:** q70 (774kbps, 536×958, 538KB) вместо q20 (155kbps, 108KB)
**Без аудио** — инстаграм отдаёт отдельным m78 URL (92kbps, можно смержить с ffmpeg)

**Скрипт:** `/root/.openclaw-instagram/download_reel.py` (для новых шорткодов)

## 2026-05-21T05:00 — TikTok Re-encode

**TikTok не принимает:**
- VP9 codec
- Нет аудио
- Разрешение меньше 1080×1920

**Решение:**
1. Через Playwright достаём m78 URL (аудио-дорожка отдельно от инстаграма)
2. ffmpeg: combine video (q70/VP9) + audio (m78/AAC) → H.264 + AAC @ 1080×1920

**Команда:**
```bash
ffmpeg -i reel_hd.mp4 -i reel_audio.mp4 \
  -c:v libx264 -profile:v main -preset fast -crf 23 \
  -vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2,setsar=1,fps=30" \
  -c:a aac -b:a 128k -shortest \
  -pix_fmt yuv420p -movflags +faststart \
  reel_tiktok_ready.mp4
```
